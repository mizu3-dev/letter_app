# 「手紙ポスト」アプリ Firebase移行・新機能・おしゃれUI/UX統合仕様書

本ドキュメントは、「手紙ポスト」アプリにおけるローカルモック環境からFirebaseプラットフォームへの移行設計、指定された新機能（手書きフォント作成、マイページ統合、手紙の仕様拡張、グループ機能、通知等）、および手書きワイヤーフレームに基づく情緒的で洗練された「おしゃれなUI/UX演出」を網羅した統合仕様書です。

---

## 1. アプリケーション概要と基本仕様

「手紙ポスト」は、即時性を重視する現代のチャットアプリとは対照的に、あえて手紙が届くまでに時間を要する（配送時間を要する）「スローライフ」なコミュニケーションを提供するSNSです。
デジタルでありながら、本物の紙やペンを使った手紙のような温かみ、おしゃれな質感、そして届くまでのワクワク感を演出する情緒的なUI/UXを最大の特徴とします。

### 1.1 主要機能と画面構成・情緒的演出仕様
下部ナビゲーションには5つのタブを配置し、中央の「手紙作成」ボタンは丸く大きく際立たせたモダンなUIとします。
* **配置**: `ポスト` / `レーダー` / `手紙作成`（丸く大きく配置） / `マイページ` / `設定`

#### ① ポスト（受信箱・送信履歴・配送ステータス表示）
* **情緒的な配送ステータス（「てがみをまつ」演出）**: 
    * 配送中の手紙は、無機質な秒単位のカウントダウンではなく、「約h時間後」のように大まかに表現します。
    * 画面上には、可愛い配達員や車のイラストとともに、現在の状況が「向かい中…もう少し！」「しわけ中 まだまだ」といった情緒的な言葉で可視化され、手紙を待つ時間を楽しく演出します。
* **「うけとる」・開封演出**:
    * 手紙が届くと「手紙がとどきました！」という通知とともに、画面上に味のある「ポスト」が表示されます。
    * 届いた手紙は「～さんから」と表示され、「あける」または「すてる」を選択できます。
    * 「あける」を選択すると、実際に封筒から手紙を取り出して広げるような、おしゃれな開封アニメーション演出が入ります。
* **受信手紙の描画と過去の筆跡維持（重要仕様）**:
    * 届いた手紙は、送信者が指定した便箋テーマ、文字サイズ、文字色、フォントで美しく描画されます。
    * 送信者が手紙を送信した「その時点の筆跡」を正確に維持するため、手紙ドキュメントには送信時のフォントの**バージョン情報（リビジョン番号）**を記録します（例: `fontVersion: 3`）。
    * 送信後に送信者がフォントに新しい文字を追加したり手書き文字を書き直したりした場合でも、**過去に送受信された手紙のフォントが勝手に書き換わらず、送信当時の筆跡がそのまま維持**されます。
* **送信済手紙**: 自分が送信した手紙の内容も後から確認できます。既読機能は設けず、「相手が手紙を発見したか（受け取ったか）」のみがステータスとして表示されます。
* **シェア機能**: 手紙の美しくスタイリングされた便箋イメージを、画像形式（`RepaintBoundary` を活用）でエクスポートしてシェアできます。

#### ② レーダー（置き手紙レーダー）
* フレンドが特定のGPS位置に設置した「置き手紙（Drop Letter）」を探すための機能です。距離（50m以内）と方向に基づいてレーダー風の画面上にプロットし、実際にその場所まで近づくことで発見・開封できます。

#### ③ 手紙作成（おしゃれな執筆・封緘プロセス）
入力スペースを最大限に広く確保し、本物のレターセットを選んで送るまでの一連のストーリーを五感に訴えかけるようなおしゃれなアニメーションで構築します。

1. **「かく（紙選び）」**:
    * 最初に便箋（かみえらび）画面が表示されます。本物の紙の束を「ベラベラ（パラパラ）」とめくって選ぶような、リッチな触感・視覚エフェクトUIを提供し、お気に入りのデザインを「決定」します。
2. **「かき中（エディタ）」**:
    * 文字数制限を撤廃し、自由な長文作成が可能です。
    * 画面上部に「ペンせんたく」「色せんたく」のツールバーを配置し、キーボードで入力しながら、文字やブロック単位で個別に大きさや色、フォント（既定の明朝体、ゴシック体、手書き風、または作成したオリジナルフォント）を柔軟に変更できます。
    * 手紙内で好きな場所に改ページを挿入でき、ページ（便箋）ごとに異なる便箋テーマを選択できます。
    * 手紙ごとに写真を1枚のみ添付できます（Firebase Storageへアップロード）。
3. **「かきおわり・しまう演出（封緘）」**:
    * 手紙を書き終えると「かんせい！！」画面に進み、手紙を包む「ふうとう」や、アクセントとなる「シール」等をおしゃれな選択肢から選びます。
    * その後、書いた便箋を丁寧に折りたたみ、選んだ封筒の中にシュッと「しまう演出（アニメーション）」が走り、「てがみ かんせい！」状態になります。
4. **「おくる（配送モード選択）」**:
    * 配送手段を選択する画面では、車などの可愛いアイコンとともに所要時間（距離算出：1kmあたり1分、最低5分など）が表示されます。
    * 週3回までの制限がある「速達（郵送モード）」や、特定のGPS位置に配置する「置き手紙モード」から選択し、「決定」すると手紙が送り出されます。
    * 手紙を相手に引き渡す（またはポストに投函する）ような、温かみのある「手紙わたすシーン」のアニメーションを最後に挟みます。
5. **「交換日記」拡張UI**:
    * 特定のフレンドとは、ノートを開いたような情緒的な「見開きUI」の画面で、お互いの写真やシール、テキストを貼り付け合える交換日記形式のレイアウトを選択肢として導入できるように設計を拡張します。

#### ④ マイページ（旧「フレンド」）
従来のフレンド一覧とアカウント管理（プロフィール編集）を融合した画面です。
* **アカウント情報**: 自分の表示名やフレンドコード（3〜15文字以上、バリデーションあり）を確認・編集できます。
* **フレンド追加機能**: 「フレンドID検索」「QRコードスキャン」「近距離通信（BLE / Nearby Connections 等を用いた近接端末マッチング・フレンド申請送受信UI）」をサポート。
* **グループ機能**: 複数のフレンドからなる「グループ」を作成でき、グループ全員宛ての一斉送信が可能です。
* **手書きオリジナルフォント作成機能（一人1セット制限・バージョン管理）**:
    * オリジナルフォントはユーザー一人あたり1セットのみ作成・保持可能とします。
    * まだ自分が書いたことのない文字を1文字ずつ手書きで追加登録し、フォントを少しずつ成長（文字を追加・差分成長）させていきます。
    * 文字を追加・修正してフォントファイルを更新する際、フォントのバージョン（リビジョン番号）をインクリメントし（例: `font_v3.ttf` として新規保存）、過去の手紙の表示再現性を保証します。
    * **フォント作成UI**: 縦画面の上部に対象の文字（お手本）を大きく表示し、下部に1:1の手書きキャンバス（入力欄）を配置。
    * **管理・エクスポート**: 登録済み文字一覧の確認、追加・修正、および外部へフォントファイル（TTFまたはOTF）としての書き出し。

#### ⑤ 設定
一般的な設定項目を集約した画面です。
* **カラーモード選択**: ライトモード / ダークモード / システム同期。
* **言語選択**: 日本語 / 英語 などの多言語切り替え。

### 2.2 フォントデータのストレージ＆通信量最適化設計
バージョンごとに巨大なTTF/OTFファイルを丸ごと生成してStorageに保存・通信すると、データ容量や通信量が爆発的に増大する懸念があります。これを回避するため、以下のいずれかの最適化アプローチを設計に組み込みます。

#### 【推奨案：オンデバイス・インクリメンタルビルド】
* **仕組み**: 
    * サーバー（Firestore）上には、文字ごとの軽量なベクターデータ（`vectorData`: SVGパス形式）のみを保存します（1文字あたり約500バイト〜2KB程度）。
    * 受信側が手紙をロードする際、手書きフォントが指定されていれば、送信者のグリフデータ（その手紙で使われている文字の差分、または送信者のそのバージョンにおける全グリフデータ）を極めて軽量なJSON形式で同期します。
    * 受信側の端末内（オンデバイス）で、取得したベクターデータをもとに動的フォントオブジェクト（OpenType.js等のDart移植パッケージやカスタムペインター）をリアルタイムにコンパイル・生成してテキストを描画します。
* **メリット**:
    * Storageに大量のTTFファイルを保存する必要が一切なくなり、ファイルサイズを実質ゼロに抑えられます。
    * ネットワーク通信量もJSONテキストデータのみになるため、数KB〜数十KB程度に極小化されます。

#### 【代替案：手紙専用のサブセットフォント自動生成】
* **仕組み**:
    * 送信者が手書きフォントを使用して手紙を送信する際、アプリ側（またはCloud Functions）で**「その手紙に実際に含まれる文字のみ」を抽出した極小のフォントファイル（サブセットフォント）**を自動でビルド・生成し、手紙データに添付（Storageに保存）します。
    * 日本語の全文字を含める代わりに、手紙に使われている数文字〜数十文字分しか含まないため、フォントファイルの容量は数KB〜数十KBへと大幅に削減されます。
* **メリット**:
    * 過去の手紙は、手書き文字追加などの影響を受けず、送信当時の筆跡を完全に固定して超軽量なファイルサイズで永続保存できます。
